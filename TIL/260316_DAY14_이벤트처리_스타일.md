# 이벤트 처리

### 인라인 이벤트 처리

- v-on 디렉티브로 설정
- 형식
    - `v-on:[이벤트 이름]=”표현식”`
- 단축 표현
    - `@[이벤트 이름]=”표현식”`
- 이벤트 객체가 필요한 경우
    - `@click=”test($event)”`
    - 클릭 시, 이 표현식을 쓰라는 뜻 (지금 당장 실행X, 클릭 이벤트ㅇ 발생 시에O)
    - $ → vue 내부적으로 선언되고 정의되는 property (이벤트 객체)
    - 코드 상세 예시
        - 이벤트 객체가 필요한 경우, 입력값 추출
        
        ```html
        **<input type="text" @input="changeName($event)" />**
        
        <script>
        methods: {
          changeName(**e**) { ⭐
            **this.name = e.target.value;** ⭐
        ****  }
        }
        </script>
        ```
        

### 05-01.html

```jsx
<div id="app">
	금액: <input type="text" **v-model.number="amount"** /> <br />
<button **v-on:click="balance += parseInt(amount)"**> 입금</button>
<button **v-on:click="balance-= parseInt(amount)"**> 인출</button>
<br />
<h3>계좌 잔고: [{{balance}}]</h3>
</div>
```

- 형변환: `v-model.number=”amount”`
- 클릭 이벤트: `v-on:click=”balance += parseInt(amount)”`
    - 클릭 시, 표현식을 실행한다

## 이벤트 핸들러 메서드

### 이벤트 핸들러 메서드

- vue 인스턴스의 methods 속성에 정의내린다

### 05-02.html

```jsx
<div id="app">
  금액: <input type="text" v-model.number="amount" /> <br />
  **<button @click="deposit">입금</button>
  <button @click="withdraw">인출</button> <br />**
  <h3>계좌 잔고: [{{ balance }}]</h3>
</div>

<script src="https://unpkg.com/vue"></script>
<script>
  let vm = Vue.createApp({
    name: "App",
    data() {
      return {
        amount: 0,
        balance: 0,
      };
    },
    **methods**: {
      **deposit() {**
        let amt = parseInt(this.amount);
        if (amt <= 0) {
          alert("0보다 큰 값을 예금하세요.");
        } else {
          **this.balance += amt;**
        }
      },
      **withdraw() {**
        let amt = parseInt(this.amount);
        if (amt <= 0) {
          alert("0보다 큰 값을 인출하세요.");
        } else if (amt > this.balance) {
          alert("잔고보다 많은 금액을 인출할 수 없습니다.");
        } else {
          **this.balance -= amt;**
        }
      },
    },
  }).mount("#app");
</script>
```

- 함수명만 넣어서도 호출이 가능하다
    
    → deposit, withdraw가 vue 모델의 method로 들어간다는 뜻이다
    
- 메서드 호출 시, this를 꼭 써야한다
    - this가 없다면, 지역변수/전역변수로 해석한다
- 화살표 함수 사용 불가
    - 화살표 함수에서는 this가 없기 때문에, 상위 객체의 this를 가져온다
    - 따라서 화살표 함수를 이용해서 만들면 안된다

## 이벤트 객체

| **항목 (Property)** | **의미** | **실제 사용 예시** |
| --- | --- | --- |
| **target** | 이벤트를 발생시킨 **범인** | 클릭된 버튼이 정확히 어떤 버튼인지 찾아낼 때 |
| **key** | 어떤 **키보드 키**를 눌렀나? | 엔터(Enter) 키를 눌렀을 때만 검색 기능을 실행할 때 |
| **button** | 어떤 **마우스 버튼**인가? | 왼쪽 클릭인지, 오른쪽 클릭(메뉴)인지 구분할 때 |
| **preventDefault()** | **기본 동작 멈춰!** | 태그의 고유 기능(링크 이동, 폼 제출)을 막고 싶을 때 |

## 이벤트 전파와 버블링

### 이벤트 전파의 3단계

![image.png](26%2003%2022(%EC%9B%94)%20DAY%2014%20-%20%EC%9D%B4%EB%B2%A4%ED%8A%B8%EC%B2%98%EB%A6%AC,%20%EC%8A%A4%ED%83%80%EC%9D%BC,%20TodoList%20%EC%98%88%EC%A0%9C/image.png)

- **이벤트 전파**
    - 하나의 클릭 이벤트가 DOM을 따라 이동하는 것
    - **버블링** - 자식이 클릭되면 해당하는 부모까지 클릭된 것으로 간주된다
        
        → 이 이벤트들을 어떤 순서로 처리해야 할까?
        
    - 주요 설명
        - capturing: 부모가 먼저 이벤트를 처리하는 것
        - bubbling: 자식이 먼저 이벤트를 처리하는 것
        - target: 실제 이벤트를 발생시킨 요소
    - 문제 상황
        - 자식 & 부모 별로 클릭 이벤트 핸들러가 있는 경우
        - 자식을 클릭했는데 부모까지 클릭된 것으로 간주돼서 부모의 함수까지 실행되는 경우
    - 해결 방법
        - 자식 이벤트가 부모로 전달되지 않게 중단한다
        - `@click.stop`

### 05-07.html

```jsx
<div id="app">
  **<div id="outer" @click.stop="outerClick">
    <div id="inner" @click.stop="innerClick"></div>
  </div>**
</div>
```

- 버블링의 차단
    - **event.stopPropagation 호출**
    - 부모에게 이벤트를 전파하지 않는다
    - **.stop** → stopPropagation과 동일하다

## 이벤트 수식어 ⭐

### 키코드 관련 이벤트 수식어

- 키 관련 수식어
    - up, down, left, right
    - **enter**, esc
    - space, tab, delete
    - ctrl, alt, shift, meta
    - 해당 키가 눌렸을 때만 처리할 수 있다
- `@이벤트.enter=”..”` : 엔터를 쳤을 때 이벤트 핸들러 호출

### 마우스 관련 수식어

- 수식어
    - left, right, middel 등.. 키 관련 수식어 사용 가능

### exact 수식어

- 다른 수식어와 조합해 이벤트를 등록할 때
    
    → 정확하게 일치하는 조합으로 이벤트가 일어나야 핸들러가 실행되도록 설정
    
    ```jsx
    <button **@click.exact="num=num+1"**
     **@click.ctrl.exact="num=num+10"
     @click.ctrl.alt.exact="num=num+100">**
    ```
    

# 스타일 적용

## HTML의 스타일 적용

- 스타일 적용 우선순위
    - 요소의 기본 스타일 → .test → .over → 인라인
    - (낮음) → (높음)

### 06-02.html : 스타일 객체

```jsx
<div id="app">
  **<button
    :style="style1"
    @mouseover.stop="overEvent"
    @mouseout.stop="outEvent"
  >**
    테스트
  </button>
</div>

<script src="https://unpkg.com/vue"></script>
<script>
  const vm = Vue.createApp({
    name: "App",
    data() {
      return {
        **// 인라인 스타일 객체 정의**
        **style1: {
          backgroundColor: "aqua",
          color: "black",
        },**
      };
    },
    methods: {
      // 마우스가 올라갔을 때
      overEvent() {
        **this.style1.backgroundColor = "purple";
        this.style1.color = "yellow";**
      },
      // 마우스가 벗어났을 때
      outEvent() {
        **this.style1.backgroundColor = "aqua";
        this.style1.color = "black";**
      },
    },
  }).mount("#app");
</script>
```

- **객체 기반 제어**
    - CSS 클래스 작성이 아니라, JavaScript 객체 자체를 HTML의 스타일 속성에 적용한다
    - 동적인 값 계산, 컴포넌트 독립성, 우선순위 등의 사유로 객체를 직접 만들어서 쓴다
    - 바인딩
        
        ```html
        **<button :style="style1"
        	@mouseover.stop="overEvent"
        	@mouseout.stop="outEvent">**
        ```
        
    - 주의사항 - 카멜 케이스
        
        → CSS에서는 `background-color`라고 쓰지만,
        JavaScript 객체 내부에서는 `backgroundColor`와 같이 카멜 케이스로 작성해야 한다

```jsx
</script>
```

## CSS 클래스 바인딩

- 클래스 바인딩이란?
    - 자바스크립트의 상태(데이터)에 따라 HTML 태그의 class를 넣었다 뺐다 하는 기술
- 클래스 바인딩 사용 예시
    - 버튼을 클릭하면 활성화 색상으로 바뀜 (`.active` 클래스 추가)
    - 입력값이 잘못되면 테두리가 빨간색으로 변함 (`.error` 클래스 추가)

### CSS 클래스 지정

- `v-bind:class=”…”` 또는 축약표현 `:class=”…”`
    - 클래스명 문자열을 바인딩하는 방법
- true/false 값을 가진 객체를 바인딩 하는 방법
    - 3항 연산자로 클래스 지정 제어 시 복잡함
    - 객체로 바인딩해서 사용함
        - 속성명으로 클래스명을 지정
        - 속성값으로 해당 클래스 적용 여부를 true/false로 지정

### 06-08.html

```jsx
**<style>
  .btnColor { background-color: aqua; color: black; }
  .btnAlign { text-align: center; width: 120px; }
  .btnBorder { border: khaki dashed 1px; }
</style>**

<div id="app">
  <label><input type="checkbox" **v-model="btnColor"** /> 색상</label><br />
  <label><input type="checkbox" **v-model="btnAlign"** /> 정렬, 크기</label><br />
  <label><input type="checkbox" **v-model="btnBorder"** /> 테두리</label><br />

  <hr />

  <button **:class="{ btnColor, btnAlign, btnBorder }"**>
    버튼
  </button>
</div>

<script src="https://unpkg.com/vue"></script>
<script>
  const vm = Vue.createApp({
    name: 'App',
    data() {
      return {
        // 초기값은 모두 해제 상태(false)
        **btnColor: false,
        btnAlign: false,
        btnBorder: false,**
      };
    },
  }).mount('#app');
</script>
```

- 객체 구문 바인딩
    - Vue의 객체 구문
        - {’클래스명’: 데이터변수(true/false)}
        - 예시) `:class="{ btnColor: btnColor }"`
        - key와 value가 같으면 하나로 쓸 수 있으므로, 생략해서 사용 가능하다
    - 코드 예시
        - `<button **:class="{ btnColor, btnAlign, btnBorder }"**>`
        - btnColor 변수가 true면 'btnColor' 클래스를 붙이고, false면 떼어라
        - 앞의 `btnColor`: 적용할 CSS 클래스명
        - 뒤의 `btnColor`: 적용 여부를 결정할 data 변수(true/false)
- 양방향 데이터 바인딩 (v-model)
    - 체크박스를 클릭하면 `data` 내의 `btnColor` 값이 실시간으로 `true` 또는 `false`로 바뀐다
    - Vue는 이 변화를 즉시 감지하여 `<button>`의 클래스를 추가하거나 제거합니다.
- 동작 원리
    1. 데이터 변경: btnColor 변수가 true로 바뀜.
    2. Vue의 감시: :class 안에 있는 btnColor 변수가 true가 된 걸 확인.
    3. 클래스 주입: 버튼 태그에 실시간으로 class="btnColor"를 슥 넣어줌.
    4. 스타일 적용: 브라우저가 버튼을 하늘색(aqua)으로 칠함.

### 06-09.html

```jsx
<style>
  .btnColor { background-color: aqua; color: black; }
  .btnAlign { text-align: center; width: 120px; }
  .btnBorder { border: khaki dashed 1px; }
</style>

<div id="app">
  <label><input type="checkbox" v-model="btnStyle.btnColor" /> 색상</label><br />
  <label><input type="checkbox" v-model="btnStyle.btnAlign" /> 정렬, 크기</label><br />
  <label><input type="checkbox" v-model="btnStyle.btnBorder" /> 테두리</label><br />

  <hr />

  **<button :class="btnStyle">버튼</button>**
</div>

<script src="https://unpkg.com/vue"></script>
<script>
  const vm = Vue.createApp({
    name: 'App',
    data() {
      return {
        // 관련된 상태를 하나의 객체로 구조화
        **btnStyle: {
          btnColor: false,
          btnAlign: false,
          btnBorder: false,
        },
      };**
    },
  }).mount('#app');
</script>
```

- 이전 예제에서는 `btnColor`, `btnAlign` 등을 따로따로 선언했지만, 이번에는 `btnStyle`이라는 하나의 바구니(객체)에 담았습니다
- 동작 원리 요약
    - V-Model: 체크박스를 누르면 `btnStyle.btnColor` 값이 `true`가 됩니다.
    - 객체 감시: Vue는 `:class`에 연결된 `btnStyle` 객체의 변화를 감시합니다.
    - 클래스 주입: `btnStyle` 객체 내에서 값이 `true`인 키(Key) 값들을 찾아 HTML 클래스(`class="btnColor ..."`)로 넣어줍니다.

- computed
    - 역할: 기존 데이터를 가공해서 새로운 값을 계산해낸다
    - 강점: score가 변경될 때만 다시 계산된다 (이전에 계산해둔 것을 재사용한다)
- Directive (디렉티브): HTML에 내리는 명령어
    - HTML 태그 안에 v- 로 시작하는 것들
        
        
        | **명령어** | **이름** | **역할** |
        | --- | --- | --- |
        | `v-model.number` | 양방향 바인딩 | 입력창의 값과 `score` 데이터를 동기화합니다. `.number` 수식어는 입력값을 자동으로 숫자형으로 변환해줘요. |
        | `:class` | 클래스 바인딩 | `v-bind:class`의 축약형입니다. `info` 객체의 값에 따라 클래스를 넣었다 뺐다 합니다. |
        | `v-if` | 조건부 렌더링 | 조건이 `true`면 태그를 만들고, `false`면 **DOM에서 아예 삭제**해버립니다. |
- 코드의 핵심 원리
    - 감시: `computed: info()`
        - `data`에 있는 `score` 값을 관찰하다가,
        값이 바뀌면 `{ warning: true/false }` 객체를 뱉어냄
    - 동적 클래스 바인딩: `:class="info"`
        - `info.warning`이 `true`가 되면,
        input 태그에 자동으로 `.warning` 클래스가 추가되어 배경색이 변함
- **조건부 렌더링 (`v-if="info.warning"`)**
    - 점수가 범위를 벗어나 `warning`이 `true`일 때만 `<img>` 태그가 DOM에 나타남
    - `false`라면 아예 사라집니다.

```jsx
const ts = new Date().getTime();

const vm = Vue.createApp({
  name: "App",
  data() {
    return {
      todo: "",
      todolist: [
        { id: ts, todo: "자전거 타기", completed: false },
        { id: ts + 1, todo: "딸과 공원 산책", completed: true },
        { id: ts + 2, todo: "일요일 애견 카페", completed: false },
        { id: ts + 3, todo: "Vue 원고 집필", completed: false },
      ],
    };
  },
  methods: {
    // 새로운 할 일 추가
    **addTodo() {**
      if (this.todo.trim().length >= 2) { // trim() 추가로 공백 방지
        this.todolist.push({ 
          id: new Date().getTime(), 
          todo: this.todo, 
          completed: false 
        });
        this.todo = ""; // 입력창 초기화
      }
    },
    // ID를 이용해 해당 항목 삭제
    **deleteTodo(id) {**
      const index = this.todolist.findIndex((item) => item.id === id);
      if (index !== -1) this.todolist.splice(index, 1);
    },
    // 완료 상태 토글 (true <-> false)
    **toggleCompleted(id) {**
      const index = this.todolist.findIndex((item) => item.id === id);
      if (index !== -1) {
        this.todolist[index].completed = !this.todolist[index].completed;
      }
    },
  },
}).mount("#app");
```
