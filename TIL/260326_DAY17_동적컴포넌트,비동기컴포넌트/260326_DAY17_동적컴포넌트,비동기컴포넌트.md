# 컴포넌트 심화

## 3 동적 컴포넌트

### 동적 컴포넌트

- 화면에 동일한 위치에 여러 컴포넌트를 표현해야 하는 경우 사용한다
- <component> 요소를 템플릿에 작성한다
- is 특성 바인딩
    - v-bind:is 또는 :is 를 통해 어떤 컴포넌트를 랜더링할지 결정한다
    - 이때 값은 등록된 컴포넌트 명을 대입하며, 대소문자를 구분한다
- 컴포넌트 생명주기
    - 컴포넌트 변경 시, 기존 컴포넌트는 제거되고 새로운 컴포넌트가 다시 생성된다
- `<keep-alive>`
    - 컴포넌트 상태를 유지하고 싶다면 `include` 속성 등을 사용해 캐싱할 수 있다

## 4 컴포넌트에서의 v-model 디렉티브

### 사용자 정의 v-model 만들기 ⭐⭐

- 사용자 정의 v-model?
    - 기본적으로 Vue에서 `v-model`은 `input` 같은 HTML 표준 요소에만 직접 쓸 수 있다
    - 하지만 **사용자 정의 컴포넌트**에도 `v-model`을 적용하고 싶을 때가 있다
    - 그때 "부모와 자식이 어떤 약속(규격)을 지켜야 하는가"를 배워보자
- 부모 컴포넌트
    
    ```jsx
    <child-component v-model:message="parentMessage" />
    ```
    
- 자식 컴포넌트
    - 이벤트명 형식
        - 부모 컴포넌트와 `v-model`로 동기화하기 위해서는
        **이벤트명을 반드시 `update:속성명` 형식**으로 사용해야 한다
    
    ```jsx
    {
      name: "ChildComponent",
      // 1. 모델에 해당하는 속성(props)을 반드시 정의
      props: {
        message: String
      },
      // 2. value 바인딩과 input 이벤트 emit 설정
      template: `
        <input 
          type="text" 
          :value="message"
          @input="$emit('update:message', $event.target.value)"
        />
      `
    }
    ```
    
- 데이터 흐름
    - Props: 부모로부터 `message`라는 이름으로 값을 전달받는다
    - Emit: **사용자가 입력할 때마다 `update:message` 이벤트를 발생**시켜 부모의 데이터를 갱신한다

## 5 provide, inject를 이용한 공용 데이터 사용

- vue3

### props를 통한 전달

- 컴포넌트 자식이 많아지면 연속적으로 속성을 전달해야한다
- 리액트의 **Props Drilling**의 상황과 동일하다
- 이를 해결하기 위해 **provide와 inject**를 사용한다

### provide, inject

- **provide**: 조상 컴포넌트(주로 App.vue)에서 하위 컴포넌트들에게 공유할 데이터를 선언
- **inject**: 데이터를 쓰고 싶은 자식 컴포넌트에서 필요한 데이터 이름만 골라서 가져옴
- 코드 예시
    - 부모 컴포넌트 (App.vue)
        
        ```jsx
        <template>
          <div>
            <h2>최신 인기곡</h2>
            </div>
        </template>
        
        <script>
        import { computed } from 'vue';
        
        export default {
          name: 'App',
          data() {
            return {
              songs: [
                { id: 1, title: 'Blueming', done: true },
                { id: 2, title: 'Dynamite', done: true },
                { id: 3, title: 'Lovesick Girls', done: false },
                { id: 4, title: '마리아(Maria)', done: false },
              ],
            };
          },
          // 하위 컴포넌트들에게 줄 데이터 보관함
          provide() {
            return {
              icons: {
                checked: 'far fa-check-circle',
                unchecked: 'far fa-circle',
              },
              // 반응형을 유지하기 위해 computed로 감싸서 전달 [cite: 1137, 1140]
              doneCount: computed(() => {
                return this.songs.filter((s) => s.done === true).length;
              }),
            };
          },
        };
        </script>
        ```
        
        - provide
            - 공용으로 쓸 아이콘 정보와 계산된 속성(`doneCount`)을 제공한다
        - computed
            - `doneCount`처럼 동적 값은 computed로 감싸서 넘긴다
        - 화살표 함수
            - function이 아닌, 화살표함수를 사용해야 한다 (this 때문에)
    - 자식 컴포넌트 (SongListItem.vue)
        
        ```jsx
        <template>
          <li>
            <i :class="song.done ? icons.checked : icons.unchecked"></i>
            {{ song.title }}
          </li>
        </template>
        
        <script>
        export default {
          name: 'SongListItem',
          // 부모가 provide한 'icons'라는 이름으로 데이터를 주입받습니다 [cite: 1176]
          inject: ['icons'], 
          props: ['song'], // 개별 곡 정보는 부모로부터 props로 받습니다 [cite: 1177]
        };
        </script>
        ```
        
        - 자식은 `inject`를 사용하여 부모의 데이터 중 필요한 것만 가져온다
        - `inject: [’icons’]`
            - Vue가 컴포넌트를 생성할 때, `inject`에 적힌 이름들을 확인합니다.  그 후 상위 컴포넌트 추적을 통해 해당 데이터를 찾아내서 현재 컴포넌트의 **속성**으로 합친다
            - 스크립트 영역(`methods`, `created` 등)에서는 **`this.icons`**로 자유롭게 접근할 수 있다
            - 템플릿 영역(`<template>`)에서는 `this`를 생략하고 그냥 **`icons`**라고만 써도 바로 인식되
    - 자식 컴포넌트 (SlongList.vue)
        
        ```jsx
        <template>
          <ul>
            <SongListItem
              v-for="s in songs"
              :key="s.id"
              :song="s"
            /> 
          </ul>
          <div>체크된 곡 수 : {{ doneCount }}</div>
        </template>
        
        <script>
        import SongListItem from './SongListItem.vue';
        
        export default {
          name: 'SongList',
          components: {
            SongListItem,
          },
          // 부모(App.vue)로부터 직접 전달받은 데이터
          props: ['songs'], 
          // 조상(App.vue)이 provide한 데이터를 직접 주입받음
          inject: ['doneCount'], 
        };
        </script>
        ```
        
        - **`inject: ['doneCount']`**:
            - 부모가 `provide`한 계산된 속성(`computed`)을 가져온 데이터

## 6 텔레포트

### 텔레포트 Teleport ⭐

- 텔레포트란?
    - 컴포넌트의 논리적 위치와 상관없이,
    HTML 요소를 DOM 트리의 전혀 다른 위치로 순간이동시켜 렌더링하는 기능
- 텔레포트가 필요한 이유
    - **모달(Modal)이나 팝업**
        - 화면 전체를 덮어야 하는데, 부모 요소에 `overflow: hidden`이나 `z-index` 설정이 걸려 있으면 화면에 제대로 나타나지 않을 수 있다
    - **툴팁(Tooltip)**
        - 특정 위치에 고정되어야 하는데 부모의 스타일에 영향을 받을 때
- 작동 원리
    - 텔레포트를 사용하면 컴포넌트의 코드는 원래 위치에 두면서,
    실제 렌더링되는 HTML 결과물만 지정한 다른 위치로 보낼 수 있다
    - DOM 구조: `#app` 내부에서 정의되었더라도, 실제로는 `#modal`이라는 별도의 `div` 안으로 들어갑니다.
    - 사용 영역: 컴포넌트 트리 계층과 관계없이 독립적인 UI를 제공할 때 사용합니다.
- 코드 예시
    - 버튼을 눌렀을 때 2초 동안 "처리중..."이라는 모달이 화면 중앙에 떴다가 사라짐
    - index.html
        
        ```jsx
        <body>
          <div id="modal"></div> <div id="app"></div>   </body>
        ```
        
    - src/components/Modal.vue
        
        ```jsx
        <template>
          <div class="modal">
            <div class="box">
              처리중...
            </div>
          </div>
        </template>
        
        <script>
        export default {
          name: "Modal"
        };
        </script>
        
        <style scoped>
        .modal {
          display: block; position: fixed; z-index: 1; 
          left: 0; top: 0; width: 100%; height: 100%;
          overflow: auto; background-color: rgba(0,0,0,0.4);
        }
        .box {
          display: flex; flex-direction: column;
          align-items: center; justify-content: center;
          position: absolute; left: 50%; top: 50%; 
          background-color: aqua; border: double 3px gray; 
          width: 100px; height: 80px;
          margin-top: -50px; margin-left: -50px;
        }
        </style>
        ```
        
    - App.vue
        
        ```jsx
        <template>
          <div>
            <h2>최신 인기곡</h2>
            <SongList :songs="songs" />
            <br /><br />
            
            <button @click="changeModal">Teleport를 이용한 Modal 기능</button>
        
            <teleport to="#modal">
              <Modal v-if="isModal" />
            </teleport>
          </div>
        </template>
        
        <script>
        import { computed } from 'vue';
        import SongList from './components/SongList.vue';
        import Modal from './components/Modal.vue';
        
        export default {
          name: "App",
          components: { SongList, Modal },
          data() {
            return {
              songs: [
                { id: 1, title: "Blueming", done: true },
                { id: 2, title: "Dynamite", done: true },
                { id: 3, title: "Lovesick Girls", done: false },
                { id: 4, title: "마리아(Maria)", done: false },
              ],
              isModal: false, // 모달 표시 여부 상태
            };
          },
          methods: {
            changeModal() {
              this.isModal = true; // 모달을 켬
              // 2초 뒤에 자동으로 모달을 끔
              setTimeout(() => { this.isModal = false }, 2000);
            },
          },
          provide() {
            return {
              icons: {
                checked: "far fa-check-circle",
                unchecked: "far fa-circle",
              },
              doneCount: computed(() => {
                return this.songs.filter((s) => s.done === true).length;
              })
            };
          }
        };
        </script>
        
        <style>
        /* 아이콘 폰트 임포트 */
        @import url("https://cdnjs.cloudflare.com/ajax/libs/font-awesome/5.14.0/css/all.min.css");
        </style>
        ```
        
        - changeModal(){} → 화살표함수 사용 (this 때문에)

## 7 비동기 컴포넌트

### 지금까지의 컴포넌트

- index.html을 로드하고 관련된 .js 파일을 로드한다
    
    → 필요하지 않은 컴포넌트도 미리 로딩하므로,
    초기 화면 로딩 시간이 지연되는 원인이다
    

### 비동기 컴포넌트란?

- 컴포넌트가 **화면에 필요한 시점에만 .js 파일을 로드한다**
- 프로젝트 **빌드 시, 비동기 컴포넌트용 파일이 별도로 분리되어 생성**된다

### 비동기 컴포넌트 사용 방법

- `defineAsyncComponent` 메서드 사용
    - 일반적인 임포트 (동기 방식)
        
        ```jsx
        import SyncComponent from './SyncComponent.vue';
        ```
        
    - 비동기 컴포넌트 임포트 (Lazy Loading)
        
        ```jsx
        import { defineAsyncComponent } from 'vue';
        
        const AsyncComponent = defineAsyncComponent(() => 
          import('./AsyncComponent.vue')
        );
        ```

# 클래스

### 상속이란?

- 기존의 생성자 함수나 객체를 기반으로 새로운 생성자 함수나 객체를 쉽게 만드는 것

### class 키워드를 이용한 상속 (최신 방식)

- 키워드
    - `extends`: 자식 클래스가 부모 클래스를 상속받음을 선언한다
    - `super()`: **자식 클래스의 생성자 내에서 반드시 호출**해야 하며,
    **부모의 생성자를 실행하고 `this`를 생성**한다
    - `super.메서드명()`: 부모 클래스에 정의된 메서드를 호출할 때 사용한다
    
    ```jsx
    class GrandChild extends Child {
      constructor(name, age, address) {
        super(name, age);           // 부모(Child)의 생성자 호출
        this.address = address;
      }
      print() {
        super.print();              // 부모의 print() 메서드 호출
        console.log("주소 : " + this.address);
      }
    }
    ```
    
- instance of 연산자
    - 상속 관계를 확인하기 위해 쓰이는 키워드
    - **`instanceof`**: 객체가 특정 생성자(또는 클래스)로부터 만들어졌는지 검사한다
        - `square instanceof Rectangle` 결과가 `true`라면,
        `square`는 `Rectangle`을 상속받은 객체라는 뜻이다

### closure를 활용한 캡슐화 (고전 방식)

- 22_inheritance.js → 데이터를 안전하게 숨기는 기법
    
    ```jsx
    function Rectangle(width, height) {
      // width와 height는 함수의 매개변수(지역 변수)
      
      // 외부에서 직접 변수에 접근할 수 없도록 메서드를 통해서만 소통
      this.getWidth = function () { return width; };
      this.getHeight = function () { return height; };
      
      this.setWidth = function (w) {
        width = w; // 매개변수 width에 접근하여 값을 수정
      };
      
      this.setHeight = function (h) {
        height = h; // 매개변수 height에 접근하여 값을 수정
      };
    }
    ```
    
    - 클로저 기법인가요?
        
        이 함수가 실행되고 나면 `Rectangle` 내부의 `width`와 `height` 변수는 원래 메모리에서 사라져야 합니다. 하지만 이 코드에서는 **클로저** 덕분에 변수가 살아남습니다.
        
        - **비공개 변수 (Private Variable)**
            - `width`와 `height`는 `this.width`가 아니라 일반 변수(매개변수)로 선언되어있다
            - 따라서 `new Rectangle()`로 만든 객체에서 `rect.width`라고 쳐봐야 `undefined`가 나온다 → **외부 접근이 차단된 것**
        - **환경의 기억**
            - `getWidth`, `setWidth` 같은 내부 함수들은 **자신이 태어난 환경인 `Rectangle` 함수의 지역 변수들을 기억한다**
            - 이 내부 함수들이 외부로 노출(객체의 메서드로 등록)되면서, 상위 함수가 종료된 후에도 `width`와 `height`에 접근할 수 있는 통로 역할을 하게 된다
- Getter/Setter 메서드
    - get, set으로 정의되는 것을 프로퍼티 라고 칭한다
