# 오전 수업

### 생명 주기

- 변수의 생명 주기
    - 일반적인 경우 (함수 스코프): 지역 변수는 함수 호출 시 생성되고 함수 실행 종료 시 소멸한다
    - **내부 함수와 엮이는 경우 (클로저): 함수가 종료되어도 내부 함수가 외부 함수의 변수를 참조하고 있다면 소멸하지 않고 유지된다**
    - 이를 클로저라고 하며 지역 변수의 생명 주기는 외부 함수보다 길어진다
- 외부 함수의 제한
    - 스코프의 보호를 받기 때문에 외부 함수에서 내부 함수에 선언된 변수에 직접 접근할 수 없다

### 익명 함수

- 이름을 가지지 않는 함수
    - 함수 자체가 값으로 취급되어 변수에 할당되거나 다른 함수의 인자로 직접 전달될 때 사용된다
    - 함수 표현식
        - 변수에 함수를 담아 변수 이름을 함수 이름처럼 사용한다
            
            ```jsx
            const sayHello = function() {
                console.log("안녕하세요!");
            };
            ```
            
    - 화살표 함수
        - 가장 현대적인 형태로, function 키워드 또한 생략한다
            
            ```jsx
            const add = (a, b) => a + b;
            ```
            

### 함수를 매개변수로 전달하기

- 함수적 프로그래밍
    - 함수를 값으로 사용 가능하다
    - 즉, 함수에 대입 연산이 가능하다
    - ex) a = b 에서, b 파트에 함수를 쓸 수 있다
    - 보통 매개변수에서도 대입 연산, return 값 전달도 대입 연산이다
    - callback 함수에서 많이 쓰이는 기법이다
- 예시 - 매개변수로 함수 전달
    
    ```jsx
    var add = function(a, b){
    	return a + b;
    }
    ```
    
    ```jsx
    function calc(a, b, **f**) {
    	return **f(a, b)**;
    }
    ```
    
    ```jsx
    console.log("2 + 3 = " + calc(2, 3, add);
    ```
    

### 함수를 리턴하는 함수

### 클로저

- 클로저란?
    - 함수가 선언될 당시의 주변 환경을 기억하는 함수
    - 내부 함수가 외부로 반환되어 어딘가에 저장되면 이 외부 함수의 환경을 박제해둔다
    - 예시
        
        ```jsx
        function createCounter() {
            let count = 0; // 외부 함수의 지역 변수 (박제될 대상)
        
            **return function() { // 내부 함수 반환
                count++; 
                return count;
            };**
        }
        
        **const myCounter = createCounter();** // createCounter는 여기서 실행 끝!
        
        console.log(myCounter()); // 1
        console.log(myCounter()); // 2
        console.log(myCounter()); // 3
        ```
        
        - `createCounter()`는 호출 직후 이미 사라졌다
        - 하지만 리턴된 내부 함수가 `myCounter`에 저장되면서, 자기가 참조하고 있는 `count` 변수를 메모리에서 지우지 못하게 꽉 붙잡고 있다
- 클로저를 사용하는 이유
    - 사용하지 않으면 count나 greeting을 **전역 변수**로 만들어야 한다
    - 클로저를 쓰면 변수는 숨기고, 특정 함수를 통해서만 그 값을 조작할 수 있게 된다
- 클로저 특징
    - 규칙 위반 가능
    
    ```jsx
    function test(name){
    	**var output = 'Hello ' + name + '...!';**
    	
    	return function(){
    		**console.log(output)**
    	} **// 함수 종료 이후에도 내부함수에서 외부함수를 접근할 수 있다 (name)**
    }
    
    test('JavaScript')();
    ```
    
    - test() 함수로 생성된 공간
        - 현재 예시에서 변수는 할당되지 않았지만, 관리되고 있다

### 클로저 - 독립된 기억 상자

```jsx
var test_1 = test('Node');
var test_2 = test('Javascript');

test_1();
test_2();
```

- **상태 보존**
    - 외부 함수 `test`가 종료된 후에도 `name` 값이 유지됨.
- **독립성**
    - test_1()은 var test_1에서 만들어진 ctx에 접근하고,
    test_2()은 var test_2에서 만들어진 ctx에 접근한다.
    - 이 둘은 똑같은 test 함수를 소스 코드로 공유하지만, 실행될 때 캡처된 환경이 각각 다르기 때문에 다른 결과를 출력한다

### Vue.js란?

기존 vanilla Js는 이벤트에 중점을 둬서 불편함이 존재했다

따라서 Data Driven이 등장했다

angular, react 등 각각 기본 철학은 같지만, 구현 메커니즘이 다르다

이 둘의 장점만 취한 것이 vue.js이다

### MVVM 패턴

- 기본 철학: 데이터가 바뀌면 작업을 하겠다
- Model - View - ViewModel
    - Model: 데이터
    - View: 출력 (HTML, 결과)
    - ModelView: 데이터가 바뀔 때 랜더링 하는 것 (데이터가 바뀌면 출력되도록 하겠다)
- 애플리케이션 로직과 UI의 분리를 위해 설계된 패턴
    - View: HTML과 CSS로 작성 → DOM Tree로 관리 (화면 랜더링)
    - ViewModel: View의 실제 논리 및 데이터 흐름 담당

### 가상 DOM

- 브라우저 메모리에서 관리하는 DOM에 대한 추상 객체
- 빠른 UI 렌더링 성능을 제공
    - 브라우저 DOM은 화면에 직접 그리는 작업을 수행
    - 가상 DOM은 변경된 부분을 탐지하는 것이 목적

핵심은 **현재 버전의 DOM과 이전 버전의 DOM이 다른 것을 어떻게 확인하냐**는 것이다

⇒ **대입 연산자가 실행되면** 값이 바뀐다고 생각한다

### 기본 파라미터와 가변 파라미터

- 기본 파라미터
    - 함수 호출 시 인수 생력했을 때 가지는 기본값을 지정
    - function func(arg1, agr2 = 기본값);
- 기본 파라미터 Rest Parameter
    - 배개변수의 개수가 달라도 된다
    - 전달하는 파라미터의 개수를 가변적으로 적용할 수 있다
    - 매개변수 앞에 … 연산자를 지정한다
        - 전달된 인수를 매개변수로 매칭한 후,
        - 매칭되지 않은 인수를 모아 배열로 지정 → 가변 파라미터로 전달
        
        function func(arg1, arg2, …argv) { … }
        
        → arguments: 함수 호출시 실제 사용된 것들로 호출된다
        
    - 1개만 지정 가능, 마지막 매개변수에만 지정 가능

```

 console.log(name, age, favoriteFoods)
```

- 02-02 블록 스코프, 스코프 체인
    
    ```jsx
    let msg = "GLOBAL";
    function outer() {
        let msg = "OUTER";
        console.log(msg);
        if (true) {
            let msg = "BLOCK";
            console.log(msg);
        }
    }
    outer();
    ```
    
    ```jsx
    OUTER
    BLOCK
    ```
    
    - 블록 스코프
        - let으로 선언된 변수는 중괄호로 감싸진 블록 내부에서만 유효하다
        - if문에서 선언된 `let msg = "BLOCK"`은 그 블록이 끝나면 사라진다
    - 스코프 체인의 탐색 우선순위
        - JS 엔진은 변수를 찾을 때, 가장 안쪽에서 시작해 밖으로 나간다
        - `if` 블록 안에서 `console.log(msg)`를 실행할 때, 엔진은 먼저 자기 블록 안에 `msg`가 있는지 찾는다
        - 이미 `let msg = "BLOCK"`을 찾았기 때문에, 굳이 더 상위인 `outer`나 `GLOBAL`까지 나가지 않고 바로 출력을 마친다
    
- 02-03 const 변수
    
    ```jsx
    const p1 = { name: 'jhon', age: 20 };
    p1.age = 22;
    console.log(p1);
    
    p1 = { name: 'lee', age: 25 };
    ```
    
    ```jsx
    { name: 'jhon', age: 22 }
    
    p1 = { name: 'lee', age: 25 };
       ^
    
    TypeError: Assignment to constant variable.
        at Object.<anonymous>
    ```
    
    - const의 재할당 금지
        - const로 선언한 변수는 값을 재할당 하는 것이 불가능하다
        - 즉, = 연산자를 사용해서 변수에 값을 집어넣으려 하면 TypeError가 발생한다
    - p1.age = 22 가 성공한 이유
        - 자바스크립트에서 객체(Object)는 참조(Reference) 타입이다
        - 즉, 객체의 주소를 수정한 것이 아니라 속성만 수정한 것이므로 const 규칙을 위반하지 않은 것으로 간주된다

    - 구조 분해 할당
        
        배열이나 객체 안에 있는 값을 개별 변수에 한 번에 담을 때 사용한다
        
        - 배열 구조 분해 할당
            - `[]` 를 사용한다
            - 인덱스 순서대로 할당한다
            - 필요 없는 값은 쉼표만 찍어서 건너뛸 수 있다
        - 객체 구조 분해 할당
            - `{}` 를 사용한다
            - 객체의 키 값과 변수명이 일치해야 한다
- 전개 연산자
    - 나머지 매개변수가 여러 값을 하나로 **모으는** 거였다면, 전개 연산자는 하나로 뭉쳐있는 배열이나 객체를 낱개로 펼치는 역할을 한다
    - 배열에서 사용할 때
        
        ```jsx
        let arr1 = [10, 20];
        let arr2 = [30, 40];
        
        // 두 배열을 합쳐서 새로운 배열 만들기
        let combined = [...arr1, ...arr2]; 
        // 결과: [10, 20, 30, 40]
        
        // 배열 복사하기
        let copy = [...arr1];
        ```
        
    - 객체에서 사용할 때
        
        ```jsx
        let user = { name: "지수", age: 24 };
        
        let updatedUser = { ...user, age: 25 };
        // 결과: { name: "지수", age: 25 }
        ```
