# 08 컴포넌트 심화

## 1 단일 파일 컴포넌트에서의 스타일

### 전역 CSS

- **다른 컴포넌트의 스타일과 충돌 피하기**
    1. **범위 CSS (Scoped CSS)**
    2. CSS 모듈
    
    | **구분** | **범위 CSS (Scoped)** | **CSS 모듈 (Module)** |
    | --- | --- | --- |
    | **작동 원리** | HTML 요소에 고유 속성(`data-v-..`) 추가  | 클래스명을 고유한 문자열로 변환  |
    | **사용법** | `<div class="child">` (그대로 사용)  | `<div :class="$style.child">` (객체 참조)  |
    | **제어 방식** | CSS 선택자 우선순위에 의존 | 자바스크립트 객체 로직으로 제어  |

### 범위 CSS **(Scoped CSS)**

- 사용하는 이유
    - 이름 중복:
        - 별개 컴포넌트에 같은 이름의 스타일을 정의하면, 나중에 로드된 CSS가 이전 CSS를 덮어버려서 **A 컴포넌트의 배경색이 갑자기 B 컴포넌트 색으로 바뀌는 사고**가 날 수 있음
- 작동 방식
    - 컴포넌트가 렌더링하는 요소에 **특성 기반의 추가적인 식별자**를 부여함
    - `<style scoped>`
        
        → 컴포넌트가 렌더링될 때 요소에 `data-v-xxxx`와 같은 고유한 식별자가 부여됨
        → 해당 컴포넌트 내부에서만 스타일이 적용되도록 격리하여 충돌 회피
        

### CSS 모듈

- CSS 스타일을 객체처럼 처리함
- `<style module>`
    - 템플릿 내에서 `$style.클래스명` 형태로 접근하여 사용
    - JS객체로 변환되기 때문에, 컴포넌트 내에서 스타일을 동적으로 제어하기에 유리함
        - 클래스 바인딩(`:class`)를 활용함
        - 따옴표 안을 자바스크립트 코드로 인식함
            
            → 따라서 변수 사용 또는 계산, 삼항 연산자 사용 가능
            
    - 예시 코드 - 동적 스타일 적용
        
        ```jsx
        <template>
          **<div :class="isActivated ? $style.active : $style.normal">
            상태에 따른 스타일 변경
          </div>**
          <button @click="isActivated = !isActivated">전환</button>
        </template>
        
        <script>
        export default {
          data() {
            return { isActivated: false };
          }
        };
        </script>
        
        **<style module>
        .active { color: blue; font-weight: bold; }
        .normal { color: gray; }
        </style>**
        ```
        

## 2 슬롯

### 슬롯 (slot)

- 슬롯이란?
    - 부모 컴포넌트가 자식 컴포넌트에게 데이터(Props)뿐만 아니라
    HTML 템플릿 정보 자체를 전달할 수 있게 하는 기능
    - **템플릿 정보: 화면에 그려질 HTML 덩어리 (태그, 스타일, 아이콘 등의 요소들을 포함함)*
- 사용하는 이유
    - 자식 컴포넌트 제작 시, 어떤 부모는 글자 옆에 아이콘을 넣고 싶어 하고,
    어떤 부모는 빨간색 강조를 하고 싶어 할 수도 있음
    - 이떄 나중에 부모가 원하는 HTML을 넣을 수 있게 자리를 비워두는 역할을 함
- 사용하는 방법
    - 자식 컴포넌트 - 바구니 준비
        
        ```jsx
        <template>
          <div>
            <input type="checkbox" />
            **<slot>기본값(부모가 아무것도 안 보내면 보임)</slot>**
          </div>
        </template>
        ```
        
    - 부모 컴포넌트 - 내용물 제작
        
        ```jsx
        <template>
          <CheckBox2>
            **<span style="color: blue;">
              <i>Vue</i> [cite: 416, 417]
            </span>**
          </CheckBox2>
        </template>
        ```
        

### 명명된 슬롯 (**Named Slot)**

- 명명된 슬롯이란?
    - 자식 컴포넌트에 여러 개의 슬롯을 둘 때 이름을 지정하는 방식
- 사용하는 방법
    - 자식: `<slot name="이름">`
    - 부모: `<… v-slot:이름 ...>`
- 활용 예시
    - 주로 헤더, 사이드바, 푸터 등 레이아웃을 구성할 때 활용

#### **범위 슬롯 (Scoped Slot)** ⭐

- 범위 슬롯이란?
    - 자식 컴포넌트의 데이터를 부모가 전달하는 템플릿 안에서 사용할 수 있게 해줌
- 사용하는 이유
    - 자식 컴포넌트는 `items`라는 목록 데이터를 가지고 있고, 부모는 이 목록을 화면에 그리고 싶어함
    - 그런데 부모마다 *"체크된 항목은 파란색으로, 안 된 건 회색으로 그려줘"* 같은 구체적인 디자인 요구사항이 다를 때 범위 슬롯을 사용함
- 사용하는 방법
    - 자식: `<slot :data="data">`
    - 부모: `v-slot:default="slotProps"`
- 코드 예시
    - 자식 컴포넌트
        
        ```jsx
        <template>
          <div v-for="item in items" :key="item.id">
            <slot **:checked="item.checked" :label="item.label**"></slot>
          </div>
        </template>
        
        <script>
        export default {
          name: 'CheckBoxList',
          props: ['items'] [cite: 800, 801]
        };
        </script>
        ```
        
    - 부모 컴포넌트
        
        ```jsx
        <template>
          <div>
            <h3>체크 여부에 따라 디자인을 다르게!</h3>
            <CheckBoxList :items="items">
              **<template v-slot:default="p2">
                <span v-if="p2.checked" style="color: blue; text-decoration: underline"> [cite: 816]
                  <i>{{ p2.label }}</i> [cite: 816]
                </span>
                <span v-else style="color: gray">{{ p2.label }}</span> [cite: 818]
              </template>**
            </CheckBoxList>
          </div>
        </template>
        
        <script>
        import CheckBoxList from './CheckBoxList.vue';
        export default {
          components: { CheckBoxList },
          data() {
            return {
              items: [
                { id: 'V', checked: true, label: 'Vue' },
                { id: 'R', checked: false, label: 'React' }
              ]
            };
          }
        };
        </script>
        ```
        

# 클래스 (생성자 함수)

## 클래스

### 함수를 사용한 객체 생성

- 함수를 사용한 객체 생성을 하는 이유
    - 기존에는 데이터를 하나 하나 객체로 만들고 배열에 담았다
        
        → 메서드 생성 시 모든 객체에 추가해야 해서 코드가 비효율적이다
        
    - 따라서, 이를 통해 호출 한번으로 정해진 구조의 객체를 만들 수 있다
- 코드 예시
    
    ```jsx
    function makeStudent(name, korean, math, english, science) {
      let student = {
        name: name,
        korean: korean,
        math: math,
        english: english,
        science: science,
    
        getSum: function() {
          return this.korean + this.math + this.english + this.science;
        },
    
        getAverage: function() {
          return this.getSum() / 4;
        },
    
        toString: function() {
          return `${this.name}\t${this.getSum()}\t${this.getAverage()}`;
        }
      };
      return student;
    }
    
    let students = [];
    students.push(makeStudent('윤인성', 90, 83, 76, 89));
    students.push(makeStudent('박찬호', 90, 83, 76, 89));
    students.push(makeStudent('류현진', 90, 83, 76, 89));
    students.push(makeStudent('이세돌', 90, 83, 76, 89));
    students.push(makeStudent('김세진', 90, 83, 76, 89));
    students.push(makeStudent('이하나', 90, 83, 76, 89));
    
    let output = 'name\t총점\t평균\n';
    for (let i in students) {
      output += students[i].toString() + '\n';
    }
    
    console.log(output);
    ```
    
- 문제점
    - `students.push`를 6번 하면, 똑같은 기능을 하는 `getSum` 함수도 메모리에 6개가 만들어짐

### 생성자 함수

- 생성자 함수란?
    - new 키워드를 사용해 객체를 생성하는 함수
    - new의 역할
        - 새로운 인스턴스를 만들고 this에 배정한다
    - this
        - 생성될 객체의 속성을 지정할 때 사용한다
        - ex) this.name = name

### 프로토타입

- 프토로타입을 사용하는 이유
    - 생성자 함수 내부에 메서드를 넣으면,
    객체를 100개 만들 때 함수도 100개가 복사되어 메모리가 낭비됨
- 프로토타입이란?
    - 생성자 함수로 만든 객체들이 공통으로 가지는 공간
    - 메서드를 프로토타입에 딱 한 번만 정의해두면, 모든 객체가 이를 공유해서 씀
- 코드 예시
    
    ```jsx
    // 속성만 내부 정의
    function Student(name, ...) { this.name = name; }
    
    // 메서드는 외부 프로토타입에 정의
    **Student.prototype.getSum = function() { ... };**
    ```
    
- 작동 원리 - 상속
    - `Student`로 만든 객체는 자신에게 직접 `getSum`이 없더라도,
    자신의 부모 격인 `Student.prototype`을 찾아가 메서드를 실행함
